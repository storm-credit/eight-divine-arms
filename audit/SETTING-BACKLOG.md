# 《팔대신병록》 설정 백로그

상태: `active-ledger`
상위 규칙: [`operations/DESIGN-HARNESS.md`](../operations/DESIGN-HARNESS.md)
총괄: PM 오케스트레이터

이 문서는 설계 중 떠오른 아이디어를 잃지 않으면서도 현재 작업 범위를 흔들지 않기 위한 격리 장부다.

각 항목은 관련 Work Packet이 열리기 전까지 정본 문서에 직접 삽입하지 않는다.

상태:

- `captured`: 아이디어 보존
- `scheduled`: 작업 패킷 배정
- `promoted`: 본선 설계로 승격
- `merged`: 다른 항목에 통합
- `rejected`: 의도적 제외

# 백로그 운용 규칙

1. 새 아이디어는 먼저 D01~D27에 배치한다.
2. 기존 항목의 사례라면 새 상위 설정을 열지 않는다.
3. 관련 Work Packet이 열릴 때만 `promoted` 처리한다.
4. 범위에 없는 아이디어 때문에 현재 패킷을 중단하지 않는다.
5. L3 변경만 선행 Gate 재개방과 사용자 승인을 요구한다.
6. 사용되지 않은 아이디어를 억지로 본편에 넣지 않는다.
