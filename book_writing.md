# 《팔대신병록》 집필·설계 최상위 정책

상태: `provisional-canon`
역할: 저장소 최상위 운영 정책
총괄: PM 오케스트레이터

이 문서는 《팔대신병록》 프로젝트의 모든 설계·검증·집필 작업보다 우선한다.

## 1. 기본 원칙

- 본문 집필보다 전체 설계를 먼저 완료한다.
- 설정 아이디어는 자유롭게 발산하되 정본과 분리한다.
- 중요한 설정은 단일 초안으로 확정하지 않는다.
- 역사·연표·거리·소유권·인물 관계를 장부로 검증한다.
- 정본은 사용자 승인 없이 `canon`으로 승격하지 않는다.
- 이전 결정과 충돌하는 새 아이디어는 자동 덮어쓰지 않고 재검토 대상으로 등록한다.

## 2. 공식 작업 파이프라인

`Draft → Validate → Human Edit → Finalize`

### Draft

- 브레인스토밍과 구조 설계
- 복수 후보 생성
- 상태: `brainstorm` 또는 `draft`

### Validate

- 역사·지리·무협·경제·미스터리·감정 검증
- 악마의 변호인과 정합성 감사
- 상태: `review`

### Human Edit

- 사용자가 방향·정서·선호 판단
- PM이 이견과 위험 요약
- 수정·승인을 결정 로그에 반영
- 상태: `provisional-canon`

### Finalize

- 용어·연표·관계·문서 링크 최종 정리
- 장부 동기화
- 사용자 승인
- 상태: `canon`

## 3. 문서 위계

1. `book_writing.md`
2. `lore_index.md`
3. 최신 Gate 결정과 `design/SETTING-BIBLE-v1.0.md`
4. `operations/`
5. `design/`
6. `audit/`
7. `원고_index.md`
8. 미디어 문서

최신 사용자 승인 결정이 가장 높은 효력을 가진다.

## 4. 정본 사용 규칙

후속 설계는 다음만 전제로 사용한다.

- `canon`
- PM이 작업 기준으로 승인한 `provisional-canon`

후보·브레인스토밍 문서는 아이디어와 결정 근거로만 사용한다.

## 5. 설계 게이트

- Gate 0: 프로젝트 경계
- Gate 1: 역사·지리·세계 작동 원리
- Gate 2: 핵심 시스템·세력·팔대신병
- Gate 3: 원사건·인물·핵심 갈등
- Gate 4: 결말·전체 플롯·복선
- Gate 5: 장면·전투·문체·감사·집필 준비

현재 Gate 0~5는 모두 `resolved-provisional`이다.

## 6. 집필 단계

설정 설계 뒤 순서:

1. 전체 시놉시스
2. 6부 상세 개요
3. 장별 개요
4. 장면 카드
5. 초고
6. 구조 수정
7. 문체 수정
8. 정합성 감사

현재:

- 전체 시놉시스 초안 완료
- 6부 원고 개요 초안 완료
- 다음 단계: 장별 개요

장편 초고는 장별 개요와 장면 카드 뒤 시작한다.

## 7. 변경 관리

정본 변경 시 기록한다.

- 변경 전·후
- 변경 이유
- 영향 문서
- 장부 갱신
- 재검토 조건

중요한 변경은 `retcon`으로 표시한다.

## 8. 품질 기준

- 설정은 사건과 인물의 선택으로 드러난다.
- 클리셰는 장르 언어로 활용하되 변형한다.
- 반전은 기존 사실의 재해석을 우선한다.
- 전투는 목표·지형·상성·비용·결과를 가진다.
- 세력은 수입·물류·정보·지역 기반을 가진다.
- 초자연은 규칙·조건·대가가 있다.
- 여운을 위해 비극과 대량 사망에 의존하지 않는다.
- 한 장면의 새 핵심 개념은 최대 두 개를 권장한다.
- 신병은 새 능력으로 편하게 위기를 해결하지 않는다.
