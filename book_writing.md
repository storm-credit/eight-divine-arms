# 《팔대신병록》 집필·설계 최상위 정책

상태: `provisional-canon`
역할: 저장소 최상위 운영 정책
총괄: PM 오케스트레이터

이 문서는 《팔대신병록》 프로젝트의 모든 설계·검증·집필 작업보다 우선한다.

## 1. 기본 원칙

- 본문 집필보다 전체 설계를 먼저 완료한다.
- 설정 아이디어는 자유롭게 발산하되 정본과 분리한다.
- 중요한 설정은 단일 초안으로 확정하지 않는다.
- 역사·연표·거리·소유권·인물 관계를 장부로 검증한다.
- 정본은 사용자 승인 없이 `canon`으로 승격하지 않는다.
- 이전 결정과 충돌하는 새 아이디어는 자동 덮어쓰지 않고 재검토 대상으로 등록한다.

## 2. 공식 작업 파이프라인

`Draft → Validate → Human Edit → Finalize`

### Draft

- 브레인스토밍과 구조 설계
- 복수 후보 생성
- 아직 모순이 존재할 수 있음
- 상태 태그: `brainstorm` 또는 `draft`

### Validate

- 역사·지리·무협·경제·미스터리·감정 관점 검증
- 악마의 변호인 반대 검토
- 정합성 감사와 장부 확인
- 상태 태그: `review`

### Human Edit

- 사용자가 방향·정서·선호를 판단
- PM이 이견과 위험을 요약
- 사용자의 수정·승인 내용을 결정 로그에 반영
- 상태 태그: `provisional-canon`

### Finalize

- 용어·연표·관계·문서 링크 최종 정리
- 영향받는 문서와 장부 동기화
- 사용자 승인 완료
- 상태 태그: `canon`

## 3. 오케스트레이션 순서

`브레인스토밍 → 전문 설계 → 반대 검토 → 정합성 감사 → PM 통합 → 사용자 편집·승인 → 정본 승격`

PM 오케스트레이터만 다음 단계 진입과 정본 승격안을 제출한다.

## 4. 문서 위계

1. `book_writing.md` — 최상위 정책
2. `lore_index.md` — 설정·설계 허브
3. `operations/` — 게이트 결정과 운영 규약
4. `design/` — 세계·세력·신병·인물·플롯 설계
5. `audit/` — 질문·충돌·연표·거리·소유권 장부
6. `원고_index.md` — 실제 집필 허브
7. `미디어_index.md` — 파생 미디어 허브
8. `미디어_생성.md` — 미디어 제작 규칙

충돌 시 위쪽 문서의 정책을 우선한다. 단, 최신 사용자 승인 결정이 가장 높은 효력을 가진다.

## 5. 정본 사용 규칙

후속 설계는 다음만 전제로 사용할 수 있다.

- `canon`
- PM이 명시적으로 허용한 `provisional-canon`

`brainstorm`, `draft`, `review` 상태의 문서는 아이디어·검토 자료로만 사용한다.

## 6. 설계 게이트

- Gate 0: 프로젝트 경계
- Gate 1: 역사·지리·세계 작동 원리
- Gate 2: 세력 아키텍처·팔대신병 시스템
- Gate 3: 과거 진실·인물·핵심 갈등
- Gate 4: 결말·전체 플롯·복선
- Gate 5: 장면 규칙·감사·집필 준비

선행 게이트의 미해결 문제가 후속 설계를 무너뜨릴 정도라면 진행을 중지하고 이전 게이트를 다시 연다.

## 7. 집필 개시 조건

다음이 승인되기 전에는 장편 본문을 정식 집필하지 않는다.

- 작품 헌법
- 역사·세계 골격
- 핵심 세력과 신병 시스템
- 과거 진실과 핵심 인물
- 선택된 결말과 전체 플롯
- 복선·연표·거리·소유권 장부
- 장면·전투·문체 규칙

시험 장면과 문체 실험은 `sandbox` 성격으로 작성할 수 있으나 정식 원고로 간주하지 않는다.

## 8. 변경 관리

정본 변경 시 반드시 기록한다.

- 변경 전 내용
- 변경 후 내용
- 변경 이유
- 영향받는 문서
- 장부 갱신 여부
- 재검토 조건

중요한 변경은 `retcon`으로 표시하고 조용히 덮어쓰지 않는다.

## 9. 품질 기준

- 설정은 사건과 인물의 선택으로 드러나야 한다.
- 클리셰는 장르 언어로 활용하되 최소 한 요소를 변형한다.
- 반전은 기존 사실의 재해석을 우선한다.
- 전투는 목표·지형·상성·비용·결과를 가진다.
- 세력은 수입·물류·정보·지역 기반을 가진다.
- 초자연 현상은 규칙·조건·대가가 있다.
- 여운을 만들기 위해 비극과 대량 사망에 의존하지 않는다.

## 10. 현재 실행 상태

- 운영 하네스·설정 완전성 감사: `completed`
- Gate 0: `resolved-provisional`
- Gate 1: `resolved-provisional`
- Gate 2: `resolved-provisional`
- Gate 3: `in-progress`
- Gate 4: `blocked`
- Gate 5: `blocked`

현재 실제 작업은 원사건·최초 여덟 당사자·현재 팔대신병·핵심 인물·현재 갈등을 설계하는 Gate 3다.

진행 현황의 단일 기준은 `operations/PM-PROGRESS-DASHBOARD.md`, Gate 3 범위 기준은 `operations/GATE-3-SCOPE-FREEZE.md`다.
