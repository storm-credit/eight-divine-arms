# 1. Gate 2 목적

다음 후속 설계가 흔들리지 않도록 세계의 핵심 작동 시스템을 확정한다.

- 무공과 기환의 상한
- 부상·의술·독의 비용
- 병기·법구·신병의 구분
- 기록·증거·위조·권리 체계
- 팔대신병의 상위 구조
- 강호와 세력의 분류·권력·생존 기반
- 명성·서열·대회 등 강호 평가 제도의 뼈대
- 영물·보패·부적·진법·강시 등 파생 기환 요소의 경계

Gate 2는 개별 인물·신병·문파 명단을 만드는 단계가 아니다.

# 2. 현재 Gate에서 완전 설계할 분야

## C1. 강호의 제도적 경계

답해야 할 질문:

- 강호인은 법적으로 무엇인가?
- 문파·세가·무관·산장·방회·단맥전승은 무엇이 다른가?
- 사문 보증·파문·비무·복수·중재는 어떻게 작동하는가?
- 무림맹의 판정과 관부의 판결은 언제 충돌하는가?

관련 분야: D05

## C2. 정보·기록·증거

답해야 할 질문:

- 정보는 누가 어떤 속도로 전달하는가?
- 공문·족보·비급·계약·화물표·증언·물증·신병 기억의 신뢰도는?
- 위조·검열·오보·매수는 어떻게 작동하는가?

관련 분야: D07·D17

## C3. 무공 상위 체계

답해야 할 질문:

- 내공·외공·경공·점혈·호신강기·진법은 무엇을 할 수 있는가?
- 수련 속도와 성장의 한계는?
- 협공·지형·독·화기·군대가 고수를 어떻게 제한하는가?

관련 분야: D09

## C4. 의술·독·부상·회복

답해야 할 질문:

- 외상·내상·중독·감염·주화입마·정신 침식은 어떻게 다른가?
- 회복 시간과 후유증은?
- 영약과 해독은 어디까지 가능한가?

관련 분야: D11

## C5. 기환 공통 상한법

답해야 할 질문:

- 기·기억·감정·장소·시신·생물에 흔적이 남는 조건은?
- 초자연 현상은 얼마나 희귀한가?
- 실제 현상과 독·환각·기관·사기의 경계는?
- 사용 조건·대가·실패 방식은?

관련 분야: D12

## C6. 병기·유물 계층

답해야 할 질문:

- 일반 병기·명병·보병·법구·보패·신병은 어떻게 다른가?
- 제작·수리·파손·운송·감정·위조는 어떻게 작동하는가?

관련 분야: D10·D13·D22

## C7. 팔대신병 상위 시스템

답해야 할 질문:

- 팔대신병은 무엇인가?
- 왜 여덟 개인가?
- 기억·감정·동작 흔적은 어떻게 남는가?
- 진품·위조품·대체품·파손품의 지위는?
- 소유권·보관권·사용권·계승권은 어떻게 다른가?
- 여러 신병이 모이면 무엇이 가능해지는가?

관련 분야: D22

## C8. 세력 아키텍처

답해야 할 질문:

- 구파일방과 오대세가는 명단인가 정치 의석인가?
- 정파·사파·신교·혈교·황실·경제·정보·외부 무림의 경계는?
- A/B/C등급 세력의 설정 밀도는?
- 각 세력은 무엇으로 생존하고 무엇을 원하는가?

관련 분야: D08

# 3. 현재 Gate에서 뼈대만 설계할 분야

다음은 독립적인 상세 시스템으로 확장하지 않고, 용어·상한·배치 시점만 확정한다.

## F1. 부적·법구·보패·진법·풍수·점복

- 팔대신병과의 격차
- 준비 시간·재료·지형·인원 의존성
- 실제 효과와 심리·기관 효과의 혼합
- 개별 목록은 후속 Gate

## F2. 영물·영수·영초·신수·사방신

- 희귀도와 자연성
- 상징과 실제 생물의 구분
- 채집·거래·밀렵·보호
- 개별 도감은 플롯 필요 시

## F3. 강령술·잔념·강시·시귀

- 완전 부활 금지
- 의학적·기관적·기환적 원인의 분류
- 사회적 장례·검시·봉인 기능
- 개별 사건은 후속 Gate

## F4. 종교·민간신앙

- 불문·도문·교단·민간 사당의 사회적 기능
- 실제 효과와 신앙의 차이
- 사교 판정과 국가·강호 정치

## F5. 명성·서열·대회·후기지수

- 복수 평가체계
- 공식·사설·지역 명부 차이
- 평가 제외·은거기인·오보
- 대회의 기능 유형
- 실제 명단과 우승자는 후속 Gate

## F6. 생활·신분·가족·구휼

- 여성·하층민·아이·노인·부상자·외지인의 기본 위치
- 혼인·상속·양자·사승의 사회적 효과
- 구휼과 착취의 주요 주체
- 세부 생활 묘사는 인물·장면 단계

## F7. 외부·해상·변경 무림

- 중원식 정사마 분류가 통하지 않는 원칙
- 통역·교역·혼인·인질·관할
- 실제 핵심 외부 세력은 권역·플롯 확정 뒤

# 4. Gate 2 범위 밖

다음은 이번 Gate에서 확정하지 않는다.

- 실제 구파일방 최종 명단
- 실제 오대세가 최종 명단
- 개별 지역 문파·세가의 완전 설정
- 팔대신병의 최종 이름·외형·재료·현 소유자
- 실제 천하십대고수·후기지수 명단
- 무림대회 참가자·우승자·사건
- 개별 영수·영초·보패·부적·진법·강시 도감
- 주인공·동료·적대자
- 과거 원사건의 최종 진실
- 결말·전체 플롯·복선 배치
- 정식 원고와 장면 집필

이 항목은 백로그에서 보존하되 선행 패킷 완료 전 상세화하지 않는다.

# 5. 작업 패킷 순서

Gate 2는 다음 순서로 진행한다.

## Foundation Track

### WP-201 강호의 제도적 정체

주 질문:

강호·문파·세가·사문 보증은 국가 법과 어떤 관계로 존재하는가?

완료 효과:

세력 분류와 인물 신분의 기반.

### WP-202 무공 존재론과 전투 상한

주 질문:

이 작품의 무공은 무엇을 할 수 있고 무엇을 할 수 없는가?

완료 효과:

신병·진법·군대·화기·영약 균형의 기반.

### WP-203 부상·의술·독·회복

주 질문:

무공과 신병 사용이 남기는 비용은 어떻게 진단·치료·누적되는가?

완료 효과:

전투 결과와 장기 연속성의 기반.

### WP-204 기환 공통 상한법

주 질문:

팔대신병을 포함한 설명 불가능한 현상에 공통으로 적용되는 최소 규칙은 무엇인가?

완료 효과:

부적·영수·강시 등 파생 설정의 무한 확장을 차단.

## Object and Evidence Track

### WP-205 병기·법구·보패·신병 분류

주 질문:

강호에서 희귀 물건의 격과 명칭은 무엇으로 구분되는가?

### WP-206 기록·증거·감정·위조 체계

주 질문:

무림과 관부는 무엇을 진실·진품·소유권의 증거로 인정하는가?

### WP-207 팔대신병 정의와 ‘왜 여덟인가’

주 질문:

팔대신병의 수와 범주는 어떤 기원 구조에서 필연성을 얻는가?

### WP-208 신병의 기억·감정·동작 흔적

주 질문:

신병이 보존하는 흔적은 어떻게 생성·접근·왜곡·소실되는가?

### WP-209 신병 권리·계승·집합 현상

주 질문:

신병의 소유·사용·보관·계승과 복수 신병의 상호작용은 어떻게 작동하는가?

## Faction Track

### WP-210 세력 분류와 대표 의석

주 질문:

구파일방·오대세가를 포함한 강호 세력의 제도적 서열은 무엇인가?

### WP-211 세력 권력·경제 프로필

주 질문:

본편 핵심 세력에 필요한 기능 슬롯과 생존 기반은 무엇인가?

### WP-212 세력–신병 관계 행렬

주 질문:

각 세력은 신병·기록·증인·권리 중 무엇을 왜 원하는가?

## Framework Track

### WP-213 기환 하위 현상 경계

주 질문:

부적·보패·진법·영물·영수·영초·강령·강시·시귀를 어떤 상위 규칙 아래 배치할 것인가?

### WP-214 명성·서열·대회 제도

주 질문:

강호는 무인의 실력·공적·위험·잠재력을 누가 어떻게 평가하고 유통하는가?

### WP-215 생활·신분·외부 무림 최소 골격

주 질문:

후속 인물·플롯 설계에 필요한 사회생활과 비중원 집단의 최소 전제는 무엇인가?

# 6. 각 패킷의 의무 절차

모든 WP는 다음을 거친다.

1. 선행 정본·미해결 질문 확인
2. 담당 발산 에이전트 최소 3개 구조안 생성
3. 관련 분야 전문가 교차 검토 2회
4. 역사·경제·무협·독자 경험 중 필요한 검증
5. 악마의 변호인 반론
6. 정합성 감사
7. PM 통합안
8. 사용자에게 중요한 선택만 제시
9. 승인 내용과 반려안 기록
10. 관련 문서·백로그·장부 동기화

하나의 패킷에서 핵심 질문은 하나만 해결한다.

# 10. 범위 변경 규칙

새 설정이 발견되면:

- D01~D27의 하위 사례: 백로그 등록 후 해당 WP에서 처리
- 현재 WP의 핵심 누락: PM 승인 후 현재 범위에 편입
- 다른 WP 둘 이상에 영향: L2 영향 분석
- 작품 헌법·시대·신병 정의 변경: L3, Gate 재개방과 사용자 승인

현재 범위 동결 뒤에는 새로운 개별 이름과 도감을 만들지 않는다.
