# 4. 병장기 작업이 들어간 이유

사용자가 제기한 병장기별 무공과 비주류 무기는 기존 설정 분야 밖의 새 대주제가 아니다.

다음 기존 항목의 필수 교차점이다.

- WP-202: 무공이 무엇을 할 수 있는가
- WP-205: 일반 병기·명병·법구·신병의 분류
- WP-210~211: 세력별 무공 문화와 생존 기반
- Gate 3: 인물의 체격·경력·상처·주무기
- Gate 5: 전투 장면 설계 규칙

따라서 `WP-202B`로 하위 분할했으며 전체 주 패킷 수는 늘리지 않았다.

## WP-202B 현재 상태

완료:

- 병장기 구조안 4개 비교
- 공통 신체 원리와 무기 계통의 분리
- 장병·단병·방호·사격·제압 역할
- 비주류 무기의 기능적 채택 원칙
- 군진·지형·물류 관점 검토

남음:

- WP-202 무공 상한과 통합
- WP-205 병기 계층과 용어 동기화
- 최종 사용자 승인 지점 정리

# 5. 자동 진행 순서

현재부터 순서를 임의로 바꾸지 않는다.

1. WP-202A 경지·숙련·평판 언어 완료
2. WP-202B 병장기 무공을 WP-202에 통합
3. WP-202 전투 상한·경공·점혈·호신강기·군대 대응 확정
4. WP-203 부상·의술·독·회복
5. WP-204 기환 공통 상한법
6. WP-205~209 병기·증거·팔대신병 시스템
7. WP-210~212 세력 아키텍처
8. WP-213~215 파생 기환·명성·생활 골격
9. Gate 2 전체 충돌 감사
10. 사용자 승인 후 Gate 3 진입

새 아이디어는 현재 주 패킷을 직접 막는 누락일 때만 하위 작업으로 편입한다. 그렇지 않으면 백로그에 등록하고 예정 패킷에서 처리한다.

# 6. 앞으로의 진행 보고 형식

모든 주요 작업 보고에는 다음 여섯 항목을 반드시 포함한다.

## A. 전체 위치

- 현재 Gate
- 현재 주 패킷
- 전체 진행 추정

## B. 이번에 완료한 것

- 새로 확정된 규칙
- 검토·반려한 후보
- 변경된 문서

## C. 새로 추가되거나 발견된 것

- 기존 범위 안의 하위 항목인지
- 새로운 주 패킷인지
- 백로그인지
- 전체 일정에 미치는 영향

## D. 현재 작업 중인 것

- 어느 전문가가 무엇을 검토하는지
- 아직 미해결인 질문

## E. 남은 작업

- 현재 패킷의 잔여 조건
- Gate 내 남은 패킷 수
- 다음 패킷

## F. 위험·사용자 결정

- 설정 누락 위험
- 과잉 설계 위험
- 작품 취향상 사용자 선택이 필요한 지점

“완료했다”는 문장만 단독으로 사용하지 않고, 위 내용을 함께 보고한다.

# 7. PM 변경 통제

## 현재 계획에 영향을 주지 않는 추가

- 기존 패킷의 필수 하위 질문
- 이미 예약된 설정 분야의 사례
- 이름·별호·개별 무기 후보

처리:

- 하위 패킷 또는 백로그
- 주 패킷 수와 Gate 순서 유지

## 계획을 변경하는 추가

- 새로운 상위 설정 분야
- 저강도 기환을 고도 판타지로 바꾸는 변경
- 팔대신병 정의 자체를 뒤집는 변경
- 주 무대와 시대 변경

처리:

- 현재 작업 중단
- 영향 분석
- 선행 Gate 재개방
- 사용자 승인
