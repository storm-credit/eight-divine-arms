# 《팔대신병록》 오케스트레이션 운영 규약

상태: `provisional-canon`
총괄: PM 오케스트레이터

## 1. 운영 목적

이 프로젝트는 한 번의 아이디어 생성으로 설정을 확정하지 않는다.

모든 설계는 다음 순서를 거친다.

`브레인스토밍 → 전문 설계 → 반대 검토 → 정합성 감사 → PM 통합 → 사용자 승인 → 정본 승격`

## 2. 에이전트 구성

### A. 총괄 PM 오케스트레이터

책임:

- 단계와 우선순위 결정
- 과업 분해와 담당 배정
- 산출물 형식 통일
- 의견 충돌 조정
- 승인·보류·폐기 판정
- GitHub 정리와 상태 관리

PM만 `canon` 승격안을 제출할 수 있다.

### B. 주제·콘셉트 설계 에이전트

책임:

- 작품 핵심 질문과 독자 약속
- 장르 비율과 차별점
- 결말 정서와 주제 일치성
- 클리셰 변형 방향

검토 기준:

- 한 문장으로 설명 가능한가
- 팔대신병과 주제가 분리되지 않는가
- 설정이 아니라 이야기의 선택으로 드러나는가

### C. 역사·지리 검증 에이전트

책임:

- 명나라 기준 연도와 제도
- 교통·거리·경제·군사·변경
- 실제 역사와 가상 사건의 접합

검토 기준:

- 실제 역사 결과를 바꾸지 않는가
- 시대착오가 없는가
- 무림이 국가 전체를 지배하는 것처럼 보이지 않는가

### D. 무협 시스템 설계 에이전트

책임:

- 무공 체계와 고수 위계
- 문파·세가·강호 관습
- 비무·원한·사승·명분
- 전투 비용과 한계

검토 기준:

- 초자연 능력과 무공의 경계가 보이는가
- 강자가 모든 문제를 해결하지 않는가
- 세력별 무공이 이름만 다른 동일 기능은 아닌가

### E. 세력·경제 설계 에이전트

책임:

- 세력 아키텍처
- 생존 기반·재정·물류
- 동맹·경쟁·종속 관계
- 상단·표국·정보망·관부

검토 기준:

- 세력은 무엇을 먹고 사는가
- 전쟁하지 않고 거래할 이유가 있는가
- 내부 파벌과 약점이 존재하는가

### F. 팔대신병·미스터리 설계 에이전트

책임:

- 신병의 기원·기능·대가
- 과거 원사건과 현재의 오해
- 단서·거짓 단서·재해석
- 소유권 이동과 위조품

검토 기준:

- 신병이 만능 열쇠가 아닌가
- 여덟 개가 기능적으로 중복되지 않는가
- 반전이 새로운 정보 투척이 아니라 기존 사실의 재해석인가

### G. 인물·감정 설계 에이전트

책임:

- 주인공의 결핍·욕망·오해
- 동료·경쟁자·적대자
- 선택의 대가와 관계 변화
- 결말의 감정적 만족과 여운

검토 기준:

- 인물이 설정 설명 도구로만 쓰이지 않는가
- 갈등이 오해 하나로만 유지되지 않는가
- 결말의 대가가 주인공의 핵심 욕망과 연결되는가

### H. 서사 구조 설계 에이전트

책임:

- 결말 역산
- 부·권·장·장면 구조
- 발견 순서와 전환점
- 사건·인물·미스터리의 동시 진전

검토 기준:

- 각 부에 독립적인 목표와 결산이 있는가
- 팔대신병 수집이 반복 퀘스트가 되지 않는가
- 중반 이후 긴장이 단순히 더 강한 적으로 상승하지 않는가

### I. 악마의 변호인 에이전트

새 설정을 만들지 않는다.

책임:

- 뻔함·편의성·작위성 지적
- 독자가 예상할 반전 추적
- 설정의 허점과 감정적 불쾌감 검사
- 더 단순한 대안 제시

### J. 정합성 감사 에이전트

새 설정을 만들지 않는다.

책임:

- 연표·연령·거리·소유권 감사
- 용어·등급·관계 중복 검사
- 정본과 초안 혼재 방지
- 미해결 질문 장부 관리

## 3. 사고 모드 배정

실제 모델 이름보다 과업에 맞는 사고 모드를 사용한다.

- 발산 모드: 브레인스토밍, 후보 생성, 변형안 탐색
- 심층 추론 모드: 세계 규칙, 인과관계, 결말 역산
- 사실 검증 모드: 역사·지리·제도 확인
- 비판 모드: 클리셰, 허점, 과잉 설정 검사
- 편집 모드: 중복 제거, 용어 통일, 문서화

하나의 산출물은 최소 세 가지 모드를 거친다.

## 4. 단계별 투입 원칙

### 프로젝트 헌법

주제·콘셉트 → 역사 검증 → 악마의 변호인 → PM

### 역사·세계 작동 원리

역사·지리 → 세력·경제 → 무협 시스템 → 감사 → PM

### 세력 아키텍처

세력·경제 → 무협 시스템 → 역사 검증 → 악마의 변호인 → PM

### 팔대신병 시스템

신병·미스터리 → 주제·콘셉트 → 무협 시스템 → 인물·감정 → 감사 → PM

### 인물과 플롯

인물·감정 → 서사 구조 → 신병·미스터리 → 악마의 변호인 → PM

## 5. 승인 게이트

### Gate 0 — 프로젝트 경계

장르, 역사, 초자연, 정서, 금지선 승인.

### Gate 1 — 세계 골격

역사·지리·권력·경제·강호 관습 승인.

### Gate 2 — 핵심 시스템

세력 아키텍처와 팔대신병 시스템 승인.

### Gate 3 — 서사 엔진

과거 진실, 인물 욕망, 핵심 갈등 승인.

### Gate 4 — 결말과 플롯

결말, 전체 플롯, 복선 구조 승인.

### Gate 5 — 집필 준비

장면 규칙과 감사 장부 승인.

## 6. 상태 규칙

- `brainstorm`: 자유 아이디어. 충돌 허용.
- `draft`: 구조를 갖춘 제안.
- `review`: 전문 검토 중.
- `provisional-canon`: 후속 설계의 임시 전제.
- `canon`: 사용자 승인까지 완료된 정본.
- `archive`: 폐기하지 않고 보존하는 탈락안.

후속 문서는 `canon` 또는 명시된 `provisional-canon`만 전제로 삼는다.

## 7. PM 출력 형식

각 단계의 PM 통합안은 다음을 포함한다.

1. 이번 단계의 목적
2. 확정 가능한 항목
3. 보류 항목
4. 전문가 간 주요 이견
5. 위험과 수정 조건
6. 다음 단계 진입 조건
